# EPIC-003: QBD/MAP/PH-стек (матрично-аналитические методы)

- **Статус:** done (2026-07-08)
- **Создан:** 2026-07-08
- **Roadmap:** [../roadmaps/qbd_map_ph_roadmap.md](../roadmaps/qbd_map_ph_roadmap.md)

## Цель

Флагманское направление: поддерживаемый Python-стек матрично-аналитических методов —
PH/MAP-распределения, QBD-решатель, калькуляторы M/PH/1, PH/PH/1, MAP/PH/1. Закрывает
моделирование коррелированного/bursty трафика, где пуассоновские модели ошибаются
на порядки.

## Контекст

Наивысшая научная ценность по [gap-анализу](../models_gap_analysis.md); ниша свободна
(BuTools заморожен с 2016, SMCSolver — академический MATLAB); прямой фит в научный фокус
проекта (retrial+MAP) и в имеющуюся фазовую машинерию Такахаси-Таками. QBD-ядро заодно
даёт альтернативный точный решатель для кросс-валидации M/H2/n.

## Результаты

- `most_queue/random/map_ph.py` — `PHDistribution` (моменты, pdf/cdf, сэмплинг CTMC,
  конвертеры Exp/Erlang/H2/Cox) и `MAP` (интенсивность, моменты интервалов, лаг-корреляции,
  сэмплинг с сохранением фазы; фабрики Poisson / PH-renewal / MMPP). 9 юнит-тестов.
- `most_queue/theory/matrix/` — QBD-решатель (logarithmic reduction, граничный уровень,
  контроль sp(R) и невязки) и `MapPh1Calc`/`MPh1Calc`/`PhPh1Calc`.
- Валидация вместо BuTools (заморожен, без лицензии) — по точным классикам и симуляции:
  M/PH/1 = M/M/1 и Полячек–Хинчин (1e-4), PH/PH/1 = GI/M/1 (среднее), MAP/PH/1 = симуляция
  (QsSim с MAP-источником), корреляция при равных mean/cv увеличивает E[W].
- **Бонус кросс-валидации**: обнаружен баг старших моментов `GIM1Calc`
  (зафиксирован в бэклоге EPIC-001).
- Демо-ноутбук `tutorials/map_ph_correlation.ipynb`.
- Отложено: BMAP/G/1, MAP-fitting, MAP/M/n — следующий MAP-эпик.
