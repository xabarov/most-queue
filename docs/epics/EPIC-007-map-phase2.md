# EPIC-007: MAP-стек, фаза 2 (многоканальность, фиттинг, BMAP)

- **Статус:** done (2026-07-09); BMAP/G/1 общего вида вынесен в будущий эпик
- **Создан:** 2026-07-09
- **Roadmap:** развивает [EPIC-003](EPIC-003-qbd-map-ph.md)

## Цель

Развить матрично-аналитический стек ([EPIC-003](EPIC-003-qbd-map-ph.md)) в сторону
практической применимости: многоканальные системы с коррелированным входом, подгонка
MAP по данным, пакетные марковские потоки.

## Контекст

EPIC-003 дал MAP/PH/1 (один прибор). Реальные системы с bursty-трафиком почти всегда
многоканальны (колл-центры, ЦОД), а параметры MAP приходят из измерений. Отложено в EPIC-003
именно это.

## Задачи

- [x] **MAP/M/c** (`MapMMcCalc`, `theory/matrix/map_mmc.py`) — коррелированный вход,
      c экспоненциальных приборов, QBD с уровне-зависимой границей + повторяющийся блок
      от уровня c; вероятности состояний и средние по Литтлу. Валидация (тесты
      `test_map_mmc.py`): Poisson MAP/M/c = Erlang C (1e-8), MAP/M/1 = MapPh1(exp),
      сверка с симуляцией, свойство «корреляция удлиняет ожидание», проверка
      неустойчивости — 2026-07-09. Записано в каталог (EN+RU).
- [x] **Фиттинг MAP** (`most_queue.random.map_fit`) — `fit_mmpp2` подгоняет MMPP-2 по
      (интенсивность, SCV, лаг-1 корреляция) через `least_squares` над замкнутыми формулами
      статистик MAP; `fit_map_from_trace` берёт статистики из трейса; отклоняет
      инфизибл-таргеты (SCV<1, лаг-1<0). Тесты `test_map_fit.py`: round-trip восстановление
      известных MMPP до 4 знаков, фит из трейса. Задокументировано в справочнике (EN+RU) — 2026-07-09.
- [x] **MAP/PH/c** (`MapPhCCalc`, `theory/matrix/map_phc.py`) — многоканальный с фазовым
      обслуживанием; фаза = MAP × мультимножество фаз занятых приборов, уровне-зависимая
      граница + однородный блок от уровня c. Валидация тремя точными сведениями и симуляцией
      (тесты `test_map_phc.py`): MAP/PH(exp)/c = MAP/M/c, MAP/PH/1 = MapPh1,
      Poisson MAP/H₂/c = Такахаси-Таками M/H₂/c, сверка MMPP/H₂/c с симуляцией. Каталог EN+RU — 2026-07-09.
- [x] **BMAP/M/1** (`BmapM1Calc`, `theory/matrix/bmap_m1.py`) — пакетный марковский вход,
      экспоненциальное обслуживание; M/G/1-типа цепь, решена устойчивым усечением уровней.
      BMAP-инфраструктура (`BMAPParams`, `bmap_poisson_batch`, `bmap_from_map`,
      `bmap_arrival_rate`) в `random/map_ph.py`. Валидация (`test_bmap_m1.py`): в точности =
      M^[X]/M/1 (BatchMM1) и MAP/M/1. Кросс-чек вскрыл баг `BatchMM1.get_w1` (делил на λ/μ
      вместо λ_job) — исправлен. Каталог EN+RU — 2026-07-09.
- [ ] **BMAP/G/1 общего вида** (Lucantoni, произвольное обслуживание) — отложено: нужна
      G-матрица M/G/1-типа + Ramaswami. Кандидат в отдельный эпик (фаза 3).
- [x] Каталог обновлён: MAP/M/c, MAP/PH/c, BMAP/M/1 (EN+RU) + разделы PH/MAP/фиттинг
      в справочнике распределений.

## Критерии готовности (DoD эпика)

Каждая модель — DoD «новая модель СМО» ([../DOD.md](../DOD.md)): валидация точными частными
случаями и симуляцией, запись в каталог (EN+RU) со схемой. Фиттинг — тест на восстановлении
параметров.

## Результаты

- **MAP/M/c** (`MapMMcCalc`) — многоканальный, коррелированный вход, QBD с уровне-зависимой
  границей. Валидация: Erlang C, MAP/M/1, симуляция.
- **Фиттинг MMPP** (`most_queue.random.map_fit`) — по (интенсивность, SCV, лаг-1) и из трейса.
- **MAP/PH/c** (`MapPhCCalc`) — многоканальный с фазовым обслуживанием, QBD с комбинаторным
  пространством фаз. Валидация: MAP/M/c, MapPh1, Такахаси-Таками M/H₂/c, симуляция.
- **BMAP/M/1** (`BmapM1Calc`) — пакетный вход, M/G/1-типа, усечение уровней. Валидация:
  BatchMM1, MAP/M/1.
- **Бонус кросс-валидации**: исправлен баг `BatchMM1.get_w1` (среднее ожидание завышалось ~7×).
- Каталог (EN+RU) и справочник распределений (PH/MAP/фиттинг) обновлены.

**Отложено в будущий эпик**: BMAP/G/1 общего вида (произвольное обслуживание, G-матрица
M/G/1-типа + Ramaswami).
