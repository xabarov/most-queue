# EPIC-007: MAP-стек, фаза 2 (многоканальность, фиттинг, BMAP)

- **Статус:** in progress
- **Создан:** 2026-07-09
- **Roadmap:** развивает [EPIC-003](EPIC-003-qbd-map-ph.md)

## Цель

Развить матрично-аналитический стек ([EPIC-003](EPIC-003-qbd-map-ph.md)) в сторону
практической применимости: многоканальные системы с коррелированным входом, подгонка
MAP по данным, пакетные марковские потоки.

## Контекст

EPIC-003 дал MAP/PH/1 (один прибор). Реальные системы с bursty-трафиком почти всегда
многоканальны (колл-центры, ЦОД), а параметры MAP приходят из измерений. Отложено в EPIC-003
именно это.

## Задачи

- [x] **MAP/M/c** (`MapMMcCalc`, `theory/matrix/map_mmc.py`) — коррелированный вход,
      c экспоненциальных приборов, QBD с уровне-зависимой границей + повторяющийся блок
      от уровня c; вероятности состояний и средние по Литтлу. Валидация (тесты
      `test_map_mmc.py`): Poisson MAP/M/c = Erlang C (1e-8), MAP/M/1 = MapPh1(exp),
      сверка с симуляцией, свойство «корреляция удлиняет ожидание», проверка
      неустойчивости — 2026-07-09. Записано в каталог (EN+RU).
- [ ] **Фиттинг MAP** (`fit_map` / `fit_mmpp`) — подгонка MMPP-2 / MAP-2 по (среднее, CV,
      лаг-1 корреляция) из трейса или моментов; проверка на восстановлении известных параметров.
- [ ] **MAP/PH/c** — многоканальный с фазовым обслуживанием (комбинаторное пространство фаз
      служебных приборов). Оценить объём; возможно, отдельной задачей.
- [ ] **BMAP/G/1** (Lucantoni) — пакетный марковский вход, M/G/1-тип (не QBD). Крупный кусок,
      возможно фаза 3.
- [ ] Демо/каталог: обновить модели MAP разделом про многоканальность.

## Критерии готовности (DoD эпика)

Каждая модель — DoD «новая модель СМО» ([../DOD.md](../DOD.md)): валидация точными частными
случаями и симуляцией, запись в каталог (EN+RU) со схемой. Фиттинг — тест на восстановлении
параметров.
