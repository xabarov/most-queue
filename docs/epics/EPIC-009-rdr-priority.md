# EPIC-009: RDR — общий многоканальный многоприоритетный анализ (M/PH/k, m классов)

- **Статус:** done (2026-07-10)
- **Создан:** 2026-07-09
- **Источник:** Harchol-Balter, Osogami, Scheller-Wolf, Wierman, "Multi-Server Queueing Systems
  with Multiple Priority Classes" (RDR). PDF: `works/articles/multichannel-pr/`.

## Цель

Обобщить приоритетный анализ до метода **RDR (Recursive Dimensionality Reduction)**:
многоканальные системы M/PH/k с произвольным числом m прерывающих приоритетов. Плюс быстрая
аппроксимация RDR-A и оптимизация ядра.

## Контекст: что уже есть

Ядро RDR в библиотеке **уже реализовано и провалидировано**:
- Алгоритм Нейтса (Приложение A статьи) — `theory/utils/passage_time.py` (`PassageTimeCalculation`:
  матрицы G, G_r для 3 моментов, inter-level passage times).
- Coxian-аппроксимация busy periods по 3 моментам — `fit_cox` / `CoxDistribution.get_params`.
- Две инстанции RDR: `MM2BusyApprox3Classes` (M/M/2, 3 класса — ровно Figure 3 статьи),
  `MMnPR2ClsBusyApprox` (M/M/n, 2 класса); плюс `MPhNPrty` (M/Ph/n). Все сверены с симуляцией.

Пробел до полной статьи: общий рекурсивный RDR для произвольных (m, k, PH); RDR-A; старшие моменты.

## Задачи

- [x] **Фаза 1 — оптимизация G-матрицы** (`passage_time.py`): линейная итерация
      `G = B + L·G + F·G·G` (формула (2), «slow») заменена на **logarithmic reduction**
      (`theory/matrix/qbd.py`, квадратичная сходимость) для повторяющегося уровня; для
      неповторяющихся уровней — **замкнутая форма** §A.5 (`solve(I−L−F·G⁺, B)`, один linear
      solve вместо итерации). Есть fallback на линейную итерацию, если G не стохастическая.
      Валидация: прямое сравнение G с итерацией (совпадение до 7e-11), полные результаты
      совпадают до 1e-13, все три приоритетных теста зелёные.
- [x] **Фаза 3 — RDR-A** (`RDRAPriorityCalc`): M/M/n с произвольным m прерывающих приоритетов
      через рекурсию к точному 2-классовому решателю (класс k видит агрегат классов 1..k−1 как
      один поток с Cox-2 периодом занятости). Валидация: сведение к M/M/n при m=1 (точно),
      сверка с симуляцией — равные μ (m=3,4) и **гетерогенные μ** (m=3,4): worst 5.7%.
      Тесты `test_rdr_a_priority.py`. Каталог EN+RU обновлён.
- [x] **Фаза 4 — фикс сходимости 2-классового решателя** (обнаружено при стресс-тесте RDR-A):
      критерий сходимости Такахаси–Таками в `MMnPR2ClsBusyApprox` и `MM2BusyApprox3Classes`
      отслеживал единственный скаляр `max(x)`, который для некоторых близких (μ_H, μ_L) случайно
      совпадал с произвольной затравкой `x[0]=0.4` → выход после 1 итерации на несошедшемся
      распределении (μ_H=1.2, μ_L=1.0: низкий класс 2.19 вместо 1.13). Заменён на **векторный**
      критерий (изменение всего вектора `x`) + минимум 2 итерации + кап `max_iter`. Регрессионный
      тест `test_two_class_convergence_close_mu`.
- [x] **Фаза 2 — общий RDR + аудит покрытия статьи**:
      - **Точный эталон** `MMkPriorityExact` — M/M/k с произвольным m прерывающих приоритетов,
        полная усечённая CTMC + униформизованная степенная итерация. Точен (до усечения) для
        любых (m,k) с экспоненциальным обслуживанием. Валидация: m=1 = M/M/n (closed form);
        сверяет RDR-A **без шума симуляции** — совпадение <0.5% для m=4 (Fig 5a).
      - **Аудит покрытия** ([../../works/articles/multichannel-pr/COVERAGE.md](../../works/articles/multichannel-pr/COVERAGE.md)):
        экспоненциальные модели статьи (Fig 5a, 7, 9) покрыты и корректны; PH-многоклассовая
        цель (Fig 5b/6/10) — задокументированный research-gap (нужен §2.3 с трекингом фаз
        обслуживания; переиспользование `MPhNPrty` даёт 34% ошибки для низшего класса при C²=8).
      - Общий явный RDR с программным построением QBD-блоков **superseded** точной CTMC для
        экспоненциального случая (см. решение ниже).

## Результаты

- **Оптимизация ядра RDR** (`passage_time.py`, алгоритм Нейтса): повторяющийся уровень —
  logarithmic reduction (квадратичная сходимость вместо линейной формулы (2)); неповторяющиеся
  уровни — замкнутая форма без итераций. Ускоряет масштабирование по загрузке и по размеру
  фазового пространства; на малых цепях существующих калькуляторов (фаза ~4) время съедает
  QBD-итерация базового `MGnCalc`, поэтому суммарный выигрыш там незаметен — эффект проявляется
  на больших m. Результаты бит-в-бит совпадают с прежними (rel diff 1e-13).
- **RDR-A для M/M/n, m приоритетов** (`RDRAPriorityCalc`): средние время пребывания/ожидания по
  всем классам, переиспользует провалидированный 2-классовый решатель. Точность — несколько
  процентов и при равных, и при гетерогенных μ (worst 5.7% на m=3,4). Каталог EN+RU.
- **Фикс сходимости 2-классового решателя**: векторный критерий вместо скалярного `max(x)` —
  устранил преждевременный выход итерации для близких μ (напр. μ_H=1.2, μ_L=1.0). Исправление
  распространено на `MMnPR2ClsBusyApprox` и `MM2BusyApprox3Classes`; регрессионный тест добавлен.
- **Точный эталон** `MMkPriorityExact` — M/M/k, произвольное m прерывающих приоритетов, полная
  усечённая CTMC + униформизованная степенная итерация. Точен для любых (m,k) с экспоненциальным
  обслуживанием; m=1 = M/M/n. Даёт эталон без шума симуляции: RDR-A совпал <0.5% (m=4).
  Тесты `test_mmk_prty_exact.py`. Каталог EN+RU.
- **Аудит покрытия статьи** (`works/articles/multichannel-pr/COVERAGE.md`): матрица моделей
  Fig 5–10 → реализации + измеренная точность. Экспоненциальные модели покрыты и корректны;
  PH-многоклассовая цель — задокументированный research-gap.
- **Дисперсия времени отклика (§2.4)** — `MMkPriorityExact(with_variance=True)`: точный второй
  момент по классам методом меченой заявки (absorbing-chain first-passage). Валидация: высший
  класс M/M/1 = closed form точно (8.0), tagged-среднее = Литтл до 1e-6, низкий класс = контрольная
  FCFS-resume симуляция (~1%). Первая в библиотеке точная дисперсия отклика для низких классов.
- **Робастность M/PH/k решателя** (`MPhNPrty` / базовый `MGnCalc`): (1) исправлена регрессия
  Фазы 1 (singular matrix в замкнутой форме `_G_calc` при k=1 — fallback на итерацию);
  (2) базовый Такахаси–Таками получил кап итераций + guard не-конечности/переполнения + проверку
  физичности вероятностей — вместо бесконечного цикла / молчаливого мусора (−7·10⁴⁰) теперь чистая
  ошибка. Защищает ~15 моделей на `MGnCalc`. Тесты `test_m_ph_n_robustness.py`.
- **Находка**: `PriorityQueueSimulator` возвращает прерванную заявку в конец очереди класса
  (не FCFS-resume) — среднее не меняется, старшие моменты отличаются. Задокументировано.
- **PH-цель в многоклассе (§2.3, Fig 5b/6/10) — закрыто**:
  - **`MPhPhK2Class`** — точный 2-класс M/PH/PH/k, оба класса PH. Ключ: при FCFS-resume живую
    фазу несут только ≤k заявок в обслуживании + ≤k замороженных → конечная цепь; активные заявки
    низкого класса как кортеж по возрасту. Валидация тройная: exp = `MMkPriorityExact` точно,
    high-PH/low-exp = `MPhNPrty` точно, both-PH = независимая FCFS-resume work-симуляция (~0.3%).
  - **`RDRAPriorityPH`** — RDR-A для m классов с PH-обслуживанием, использует `MPhPhK2Class`.
    Валидация: m=2 = `MPhPhK2Class` точно; m=4 all-PH C²=8 = FCFS-resume симуляция 0–1.4% по всем ρ.
  - **Находка**: для многоканальных + PH внутриклассовый порядок влияет на **среднее** (не только
    моменты), в отличие от одноканального. Реализована стандартная FCFS-resume.
  - Тесты `test_mph_ph_k_2class.py`, `test_rdr_a_priority.py`. Каталог EN+RU.

### Решение по объёму (общий RDR)

Две существующие RDR-инстанции (`MM2BusyApprox3Classes`, `MMnPR2ClsBusyApprox`) — **вручную
выведенные, жёстко закодированные** QBD-блоки под конкретные (m,k), причём m=3-код содержит
приближения (низший из «высоких» класс моделируется экспоненциально). Общий явный RDR с
программным построением блоков для произвольных (m,k,PH) — research-grade вывод с высоким риском
тонких ошибок.

Вместо хрупкого явного построения выбраны два надёжных пути, вместе дающих **общий** ответ для
экспоненциального класса моделей статьи:
- **RDR-A** (`RDRAPriorityCalc`) — метод самой статьи как практический общий: любые (m,k),
  быстрый, переиспользует проверенный 2-классовый решатель.
- **Точная CTMC** (`MMkPriorityExact`) — точный эталон для любых (m,k), экспоненциальное
  обслуживание; подтверждает RDR-A без шума симуляции (<0.5%).

Это то же инженерное решение, что в EPIC-008 (устойчивое усечение вместо хрупкой G-матрицы
Lucantoni): предпочесть корректное и проверяемое хрупкому и «наполовину верному».

### Устранённый баг сходимости (Фаза 4)

Стресс-тест RDR-A вскрыл **предсуществующий** баг 2-классового решателя: критерий сходимости
Такахаси–Таками отслеживал скаляр `max(x)`, который для некоторых близких (μ_H, μ_L) случайно
совпадал с затравкой `x[0]=0.4` и обрывал итерацию после первого шага (μ_H=1.2, μ_L=1.0 → 2.19
против 1.13 симуляции). Баг не связан с оптимизацией Фазы 1 (тот же результат при линейной
итерации G). Исправлен переходом на **векторный** критерий сходимости (по всему вектору `x`,
минимум 2 итерации, кап `max_iter`) в обоих решателях; гетерогенные μ теперь дают worst 5.7%.
