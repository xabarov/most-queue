# EPIC-017: Сети МО — закрытые сети и точные методы (MVA, Джексон, QNA, G-networks, BCMP)

- **Статус:** done (2026-07-13)
- **Создан:** 2026-07-13
- **Roadmap:** обзор сетей — [../research/queueing-networks-2026.md](../research/queueing-networks-2026.md)

## Цель

Закрыть главный разрыв между литературой по сетям МО и библиотекой: три самых цитируемых
результата теории сетей (BCMP 1975, MVA 1980, свёртка Бьюзена 1973) в most_queue не
представлены вообще — закрытых сетей нет ни в каком виде. Параллельно добавить точные
product-form решатели (Джексон, Геленбе) как эталоны для существующей приближённой
декомпозиции и апгрейдить саму декомпозицию до QNA (непуассоновские внутренние потоки).

После эпика сетевой модуль покрывает: открытые (декомпозиция + точный Джексон + QNA),
закрытые (MVA/Бьюзен + approx-MVA), мультикласс product-form (BCMP), негативные
(декомпозиция + точный Gelenbe product-form).

## Контекст

Текущее состояние: все аналитические сети — открытые, пуассоновский внешний поток, одна
приближённая декомпозиция (баланс потоков → Такахаси–Таками по узлам → Gamma-аппроксимация
ПЛС → `(I−N·Q)⁻¹`). Подробный ландшафт публикаций и ранжирование — в
[обзоре](../research/queueing-networks-2026.md); пробел «закрытые сети» помечен `should`
ещё в [models_gap_analysis](../models_gap_analysis.md) и EPIC-001, README обещает
closed networks / MVA в roadmap-тизере.

Первоисточники: Reiser–Lavenberg (MVA, JACM 1980), Buzen (свёртка, CACM 1973),
Jackson (1957/1963), BCMP (JACM 1975), Whitt (QNA 1983; RQNA 2019 — indices of
dispersion), Gelenbe (G-networks product-form, JAP 1991).

Ориентир по покрытию closed-network алгоритмов: queueing package для GNU Octave
(arXiv:2209.04220).

## Задачи

### П.1 Закрытые сети, один класс: MVA + свёртка Бьюзена

- [x] `ClosedNetworkCalc(method="mva")` (`theory/networks/closed_network.py`): точный MVA
      (Reiser–Lavenberg), включая **многоканальные станции** (точно, через маргинальные
      вероятности, Bolch §8.2) и **delay-узлы** (IS, `n=None`); метрики: пропускная
      способность, L_i, W_i, загрузки, среднее время цикла.
- [x] `method="convolution"` — свёртка Бьюзена (нормализационная константа G(N),
      load-dependent факторы для многоканальных/IS станций) + кросс-проверка с MVA.
- [x] `method="schweitzer"` — approx-MVA Schweitzer/Bard; многоканальные станции — через
      аппроксимацию Зайдмана.
- [x] `ClosedNetworkSim` (`sim/networks/closed_network.py`): событийный симулятор
      (фиксированная популяция, multi-channel FCFS + delay узлы, сидируемый).
- [x] Валидация (`tests/test_closed_network.py`, 6 тестов): MVA == Бьюзен до 1e-10
      (включая многоканальные и delay); Schweitzer в пределах 3% по X; сим vs MVA
      (центральный сервер и модель терминалов); ограничение по бутылочному горлышку.

### П.2 Точный Джексон product-form (открытая марковская сеть)

- [x] `JacksonNetworkCalc` (`theory/networks/jackson_network.py`): уравнения трафика
      (общий модуль `traffic.py`) + точные M/M/n узлы; средние точны (Little).
- [x] Валидация (`tests/test_jackson_network.py`): vs `OpenNetworkCalc` (декомпозиция) —
      **фактическая ошибка декомпозиции на марковской сети 0.29%** по v1; vs симуляция
      (5%); сводимость к M/M/n (1e-12).

### П.3 QNA/RQNA: непуассоновские внутренние потоки

- [x] `OpenNetworkCalcQNA` (`theory/networks/qna.py`): распространение c² через выход /
      разрежение / слияние (гибридные веса Уитта), узлы GI/G/n с поправкой
      Kraemer–Langenbach-Belz; неподвижная точка по c²_a.
- [ ] ~~RQNA-поправки через индексы дисперсии (Whitt–You 2019)~~ — не потребовалось:
      базовый QNA уже дал выигрыш на порядок; перенесено в резерв обзора.
- [x] Валидация (`tests/test_qna.py`): тандем M/H2(c²=4)/1, ρ=0.8 — ошибка QNA **2.4%**
      против **20.0%** у базовой декомпозиции; сводимость к Erlang C при M/M/n (1e-8).

### П.4 G-networks: точный product-form Геленбе

- [x] `GNetworkCalc` (`theory/networks/g_network.py`): точное решение Gelenbe (1991),
      M/M/1-узлы, внешние позитивные/негативные потоки по узлам, маршрутизация сигналов
      P⁻; нелинейные уравнения трафика — неподвижная точка.
- [ ] ~~Мультикласс сигналов (Gelenbe 1996)~~ — в резерв (после появления спроса).
- [x] Валидация (`tests/test_g_network.py`): vs **точная CTMC** (усечённая, 2 узла,
      сигнальная маршрутизация) — совпадение 1e-6; сводимость к Джексону без негативов
      (1e-10); монотонность по интенсивности негативов. CTMC выбрана вместо симулятора:
      v в `NegativeNetwork`-симе считается только по обслуженным заявкам, а RCS/DISASTER
      декомпозиция — другая семантика удаления, чем у Gelenbe (RCH).

### П.5 BCMP / мультиклассовый MVA

- [x] `BCMPOpenNetworkCalc` (`theory/networks/bcmp_network.py`): мультикласс product-form
      для открытых сетей, станции FCFS (exp, class-independent) / PS / LCFS-PR / IS;
      проверка BCMP-условия для FCFS. Смешанные (mixed) сети — вне scope.
- [x] `BCMPClosedNetworkCalc`: точный мультичейн-MVA (рекурсия по векторам популяций);
      одноканальные queueing-станции + IS.
- [x] Валидация (`tests/test_bcmp_network.py`, 5 тестов): одноклассовые случаи сводятся
      к Джексону (1e-12) и к MVA П.1 (1e-12); **2-классовая закрытая PS-сеть vs точная
      CTMC — 1e-8**; агрегация одинаковых классов; отказ для class-dependent FCFS.

## Критерии готовности (DoD эпика)

Общий DoD ([../DOD.md](../DOD.md)). Специфично:

- MVA и свёртка Бьюзена совпадают до машинной точности; approx-MVA сходится к точному
  при малых N.
- `JacksonNetworkCalc` и `GNetworkCalc` — точные: расхождение с симуляцией только
  статистическое; задокументирована фактическая ошибка декомпозиции против точных
  эталонов.
- QNA уменьшает ошибку декомпозиции на немарковских сетях (показано в тестах/туториале).
- BCMP в одноклассовых частных случаях сводится к Джексону/MVA.

## Вне scope (резерв — см. обзор, п. 6–8)

Сети с блокировками (конечные буферы), fork-join внутри маршрутизируемой сети /
(k,l)-fork-join, transient/time-varying сети, LQN, RL-управление.

## Результаты

Сетевой модуль расширен пятью решателями (все — `most_queue/theory/networks/`):

- **`ClosedNetworkCalc`** — закрытые сети: точный MVA (включая многоканальные станции и
  delay-узлы), свёртка Бьюзена (load-dependent), approx-MVA Schweitzer/Bard; парный
  **`ClosedNetworkSim`** (`sim/networks/closed_network.py`). MVA == Бьюзен до машинной
  точности.
- **`JacksonNetworkCalc`** — точный product-form открытой марковской сети; впервые
  измерена фактическая ошибка декомпозиции `OpenNetworkCalc` на марковской сети: 0.29%.
- **`OpenNetworkCalcQNA`** — QNA Уитта: на тандеме M/H2/1 (c²=4, ρ=0.8) ошибка среднего
  времени пребывания снижена с 20% (декомпозиция с пуассоновскими внутренними потоками)
  до 2.4%.
- **`GNetworkCalc`** — точный product-form Геленбе для сетей с отрицательными заявками
  и сигнальной маршрутизацией; сверен с точной CTMC (1e-6).
- **`BCMPOpenNetworkCalc` / `BCMPClosedNetworkCalc`** — мультиклассовый product-form
  (FCFS/PS/LCFS-PR/IS) и точный мультичейн-MVA; 2-классовая PS-сеть сверена с точной
  CTMC (1e-8).

17 новых тестов (5 файлов), пример «терминалы + сервер», каталог и руководство по сетям
(EN+RU), README-таблицы обновлены. Общий модуль уравнений трафика — `traffic.py`.
В резерв (см. [обзор](../research/queueing-networks-2026.md), п. 6–8): RQNA-поправки,
мультикласс сигналов G-сетей, mixed BCMP, многоканальные станции в мультичейн-MVA,
сети с блокировками, fork-join внутри сети.
