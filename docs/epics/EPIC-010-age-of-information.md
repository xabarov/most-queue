# EPIC-010: Age of Information (AoI)

- **Статус:** done (2026-07-10)
- **Создан:** 2026-07-10
- **Roadmap:** обзор трендов — [../research/queueing-trends-2026.md](../research/queueing-trends-2026.md)

## Цель

Добавить расчёт **свежести информации** — среднего и пикового Age of Information (AoI/PAoI) — для
базовых очередных моделей. Огромная область (IoT, 5G/6G, networked control, сенсорные сети), при
этом ни один open-source пакет не даёт готовых калькуляторов AoI с валидацией. Идеально ложится на
нишу most-queue: AoI выражается через очередные примитивы (время пребывания, интервалы отправлений),
которые уже считаются.

## Контекст

AoI в момент t: Δ(t) = t − u(t), где u(t) — время генерации самого свежего доставленного апдейта.
Средний AoI — площадь под пилообразным процессом Δ(t) на единицу времени; peak AoI (PAoI) — среднее
пиков перед каждой доставкой. Для многих систем есть closed-form:
- **M/M/1** FCFS: Δ̄ = (1/μ)(1 + 1/ρ + ρ²/(1−ρ)); PAoI-формулы известны.
- **M/D/1, D/M/1, M/M/1/1, M/M/1/2** — closed-form.
- **LCFS / preemptive-LCFS** — свежее FCFS; отдельные формулы.
- **M/G/1**, **G/G/1** — через моменты обслуживания/интервалов.

Первоисточники: Yates et al. «Age of Information: An Introduction and Survey»; серия работ по
tandem/preemptive AoI (см. [research](../research/queueing-trends-2026.md)).

## Задачи

- [x] **Ядро AoI** (`theory/aoi/`): `AoICalc` — средний AoI (M/M/1) и PAoI. Средний AoI M/M/1 =
      (1/μ)(1+1/ρ+ρ²/(1−ρ)). PAoI = E[T]+1/λ (точно для одноканальной FCFS; E[T] для M/G/1 — по
      Полячеку–Хинчину). **Находка (сим поймал ошибку памяти)**: правильная PAoI = E[T]+1/λ, а не
      растиражированная «+1»-формула. Общий средний AoI M/G/1 не выражается через моменты — только
      симуляция (подтверждено обзором литературы).
- [x] **LCFS/preemptive**: `LcfsPreemptiveAoICalc` — Δ̄ = (1/μ)(1+1/ρ), устойчив при любом ρ.
- [x] **AoI-симулятор** (`sim/aoi.py`): `AoISim` — трекинг пилообразного Δ(t), дисциплины
      FCFS / LCFS / LCFS-PR на 1+ серверах, любые распределения; сидируемый.
- [x] Валидация: все closed-form сверены с симуляцией (M/M/1 avg+PAoI, M/G/1 PAoI на H2,
      preemptive-LCFS) в пределах статпогрешности; PAoI≠E[T]+1/λ для многоканальных (out-of-order)
      — область применимости задокументирована. Тесты `test_aoi.py`.
- [x] Каталог моделей (EN+RU).
- [ ] *(опционально, будущее)* Туториал `tutorials/age_of_information.ipynb`, M/D/1 closed-form,
      многоисточниковый AoI.

## Критерии готовности (DoD эпика)

Общий DoD моделей ([../DOD.md](../DOD.md)): первоисточник, валидация closed-form + симуляцией,
записи в каталог. Специфично: для каждой модели среднее AoI и PAoI совпадают с симуляцией в пределах
статистической погрешности; воспроизведение хотя бы одной таблицы/кривой из survey.

## Результаты

- **`AoICalc`** — средний AoI (M/M/1) + PAoI (общая одноканальная FCFS, через P-K).
- **`LcfsPreemptiveAoICalc`** — средний AoI preemptive-LCFS M/M/1.
- **`AoISim`** — дискретно-событийный AoI-симулятор (FCFS/LCFS/LCFS-PR, 1+ серверов).
- Все формулы сверены с симуляцией; область применимости PAoI=E[T]+1/λ (одноканальная FCFS)
  установлена эмпирически. Каталог EN+RU. Тесты `test_aoi.py` (9 шт., зелёные).

**Заметка**: симулятор поймал неверную «книжную» формулу PAoI из памяти — правильная PAoI=E[T]+1/λ.
Это иллюстрация методологии most-queue (аналитика + парная симуляция как арбитр).
