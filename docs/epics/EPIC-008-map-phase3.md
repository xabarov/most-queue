# EPIC-008: MAP-стек, фаза 3 (BMAP/G/1 общего вида, M/G/1-тип)

- **Статус:** proposed
- **Создан:** 2026-07-09
- **Roadmap:** развивает [EPIC-007](EPIC-007-map-phase2.md) → [EPIC-003](EPIC-003-qbd-map-ph.md)

## Цель

Замкнуть матрично-аналитический стек моделью **BMAP/G/1 общего вида** — пакетный марковский
вход + **произвольное** (не только экспоненциальное/фазовое) обслуживание. Требует настоящей
машинерии цепей **M/G/1-типа** (в отличие от QBD и усечения, использованных в EPIC-007).

## Контекст

EPIC-007 дал MAP/M/c, MAP/PH/c, MMPP-фиттинг и **BMAP/M/1** (экспоненциальное обслуживание,
решён устойчивым усечением уровней). Общий BMAP/G/1 с произвольным обслуживанием усечением
не берётся: нужен G-матричный подход Lucantoni.

Ниша сохраняется: эталонный BuTools заморожен с 2016 (без лицензии), поддерживаемой
pip-устанавливаемой реализации BMAP/G/1 в open source нет.

## Задачи

- [ ] **Ядро M/G/1-типа** (`theory/matrix/mg1_type.py`): G-матрица минимальным неотрицательным
      решением `sum_k A_k G^k = 0` через функциональную итерацию (или циклическую редукцию
      Bini–Meini); стационарный вектор через рекурсию Рамасвами. Проверки: sp(G) и невязка.
- [ ] **Матрицы A_k** для BMAP/G/1: число приходов BMAP за одно время обслуживания как
      матрица (для PH-обслуживания — в замкнутой форме через `expm`; для общего G, заданного
      моментами, — численно через аппроксимацию сервиса PH или квадратуру).
- [ ] **`BmapG1Calc`** — стационарное распределение длины очереди, моменты ожидания.
      Валидация: BMAP/M/1 = `BmapM1Calc` (EPIC-007); BMAP/PH/1 vs QBD (MapPh1 при size-1);
      M^[X]/G/1 (пуассоновский batch, общий сервис) — сверка с классической формулой; симуляция.
- [ ] **BMAP-симулятор** (`sim/`): расширить генерацию MAP на эмиссию пакетов (на переходе
      через D_k выпускать k заявок), интеграция в `QsSim` для независимой валидации.
- [ ] Каталог (EN+RU) + запись в справочник (BMAP).

## Критерии готовности (DoD эпика)

DoD моделей из [../DOD.md](../DOD.md): валидация точными частными случаями и симуляцией,
записи в каталог, численные проверки ядра (sp(G), невязка G-уравнения).

## Заметки по объёму

Крупный эпик: M/G/1-тип сложнее QBD (unbounded up-jumps, upper-Hessenberg), рекурсия
Рамасвами требует аккуратной реализации. Начинать с BMAP/PH/1 (A_k в замкнутой форме),
затем обобщать на BMAP/G/1. BMAP-симулятор — предпосылка надёжной валидации.
