# EPIC-008: MAP-стек, фаза 3 (BMAP/G/1 общего вида, M/G/1-тип)

- **Статус:** done (2026-07-09)
- **Создан:** 2026-07-09
- **Roadmap:** развивает [EPIC-007](EPIC-007-map-phase2.md) → [EPIC-003](EPIC-003-qbd-map-ph.md)

## Цель

Замкнуть матрично-аналитический стек моделью **BMAP/G/1 общего вида** — пакетный марковский
вход + **произвольное** (не только экспоненциальное/фазовое) обслуживание. Требует настоящей
машинерии цепей **M/G/1-типа** (в отличие от QBD и усечения, использованных в EPIC-007).

## Контекст

EPIC-007 дал MAP/M/c, MAP/PH/c, MMPP-фиттинг и **BMAP/M/1** (экспоненциальное обслуживание,
решён устойчивым усечением уровней). Общий BMAP/G/1 с произвольным обслуживанием усечением
не берётся: нужен G-матричный подход Lucantoni.

Ниша сохраняется: эталонный BuTools заморожен с 2016 (без лицензии), поддерживаемой
pip-устанавливаемой реализации BMAP/G/1 в open source нет.

## Задачи

- [x] **Инженерное решение (2026-07-09): усечение уровней вместо G-матрицы M/G/1-типа.**
      Общий BMAP/G/1 с *произвольным* обслуживанием сводится к **BMAP/PH/1** (произвольное G
      задаётся PH-фиттингом его моментов), а BMAP/PH/1 — это CTMC над (уровень, фаза BMAP,
      фаза обслуживания), который точно решается тем же устойчивым усечением, что уже дало
      совпадение BMAP/M/1 с BatchMM1 (EPIC-007). Хрупкая машинерия Lucantoni (G-матрица +
      Ramaswami) не потребовалась. Задачи «ядро M/G/1-типа» и «матрицы A_k» сняты как излишние.
- [x] **`BmapPh1Calc`** (`theory/matrix/bmap_ph1.py`) — стационарное распределение и средние
      по Литтлу. Валидация: BMAP/PH(exp)/1 = `BmapM1Calc`, BMAP/PH/1 (size-1) = `MapPh1Calc`
      (оба точно), симуляция для Poisson-batch и коррелированного MMPP-batch. Общий сервис —
      через PH-фиттинг (`fit_h2`/`fit_cox`). Тесты `test_bmap_ph1.py`.
- [x] **BMAP-симулятор** `BmapPh1Sim` (`sim/bmap.py`) — эмиссия пакетов на переходах D_k,
      PH-обслуживание; независимая валидация калькулятора.
- [x] Каталог (EN+RU): BMAP/PH/1 добавлен.

## Критерии готовности (DoD эпика)

DoD моделей из [../DOD.md](../DOD.md): валидация точными частными случаями и симуляцией,
записи в каталог, численные проверки ядра (sp(G), невязка G-уравнения).

## Результаты

- **BMAP/PH/1** (`BmapPh1Calc`) — пакетный марковский вход + фазовое обслуживание, усечение
  уровней над (уровень, фаза BMAP, фаза обслуживания). Произвольное G — через PH-фиттинг.
- **BMAP-симулятор** (`BmapPh1Sim`) — эмиссия пакетов + PH-обслуживание.
- Валидация 4 путями: BMAP/M/1 и MapPh1 (точно), симуляция Poisson-batch и MMPP-batch.
- Каталог EN+RU обновлён.

**Ключевое решение**: устойчивое усечение уровней покрыло весь практический BMAP/G/1
(через PH-представление сервиса) без хрупкой G-матрицы M/G/1-типа и рекурсии Ramaswami —
проще, корректнее, легче валидируется. Полная G-матричная машинерия Lucantoni осталась
незадействованной; при необходимости точного решения для тяжелохвостых сервисов, плохо
приближаемых PH, её можно добавить отдельной задачей.
