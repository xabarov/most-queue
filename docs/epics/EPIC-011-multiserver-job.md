# EPIC-011: Multiserver-job (MSJ) модель

- **Статус:** proposed
- **Создан:** 2026-07-10
- **Roadmap:** обзор трендов — [../research/queueing-trends-2026.md](../research/queueing-trends-2026.md)

## Цель

Реализовать **multiserver-job** модель: заявка требует **нескольких серверов одновременно** (облачные/
GPU-джобы, запрашивающие k ядер). Это принципиально отличается от M/M/c (одна заявка — один сервер) и
является самой активной датацентр-темой сообщества (группа Harchol-Balter/Grosof/Scully). Open-source
реализаций нет — это нишевый флагман уровня RDR и отрицательных заявок.

## Контекст

MSJ: каждая заявка класса i требует `server_need_i` серверов на время обслуживания; если свободных
серверов меньше, заявка ждёт. Ключевые известные результаты:
- **Стабильность** через saturated product-form систему (Grosof, Harchol-Balter, Scheller-Wolf):
  насыщенная система имеет product-form стационар → порог стабильности и выходная интенсивность.
- **2 класса, saturated MSJ** — точный и приближённый анализ (Performance Evaluation 2023).
- **FCFS MSJ** — среднее время отклика (heavy-traffic оптимальность, приближения).
- Малые конфигурации серверов — точная CTMC над (число заявок каждого класса, занятость серверов).

Первоисточники: QUESTA 2022 (CMU), SIGMETRICS heavy-traffic, saturated-MSJ 2-class (см.
[research](../research/queueing-trends-2026.md)).

## Задачи

- [ ] **MSJ-симулятор** (`sim/`): заявки с `server_need`, дисциплина FCFS/backfilling; классы с
      разными потребностями. База для валидации.
- [ ] **Точная CTMC** для малого числа серверов k и нескольких классов server-need: стационар +
      среднее время отклика/число в системе (по Литтлу). Аналог `MMkPriorityExact` по духу.
- [ ] **Saturated MSJ**: product-form насыщенной системы → порог стабильности и выходная
      интенсивность (throughput). Валидация порога симуляцией.
- [ ] **2-класс приближение** времени отклика (FCFS MSJ) — по опубликованным exact/approx.
- [ ] Валидация: CTMC vs симуляция; порог стабильности vs симуляция; сверка с числами из статей.
- [ ] Каталог (EN+RU) + туториал.

## Критерии готовности (DoD эпика)

Общий DoD ([../DOD.md](../DOD.md)). Специфично: порог стабильности saturated-системы совпадает с
эмпирическим (симуляция при ρ→порог); среднее время отклика CTMC совпадает с симуляцией; численная
сверка хотя бы с одним примером из первоисточника.
