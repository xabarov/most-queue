# EPIC-018: Сети МО, волна 2 — блокировки, fork-join в сети, MAP-вход, иллюстрации и туториалы

- **Статус:** done (2026-07-13)
- **Создан:** 2026-07-13
- **Roadmap:** обзор сетей — [../research/queueing-networks-2026.md](../research/queueing-networks-2026.md)

## Цель

Довести сетевой модуль до практической полноты: сети с конечными буферами
(производственные линии), fork-join внутри маршрутизируемой сети, непуассоновский
(MAP) внешний поток, добор точности из резерва EPIC-017 — плюс визуальная и
образовательная часть: схемы сетевых моделей в каталоге и туториалы с инсайтами.

## Контекст

EPIC-017 закрыл закрытые сети и точные методы (MVA/Бьюзен, Джексон, QNA, G-networks,
BCMP). Из обзора остались направления с высоким практическим спросом (блокировки,
fork-join в сети — 🔥 в trends-2026 §D) и синергия с MAP/PH-стеком (сети принимают
только пуассоновский внешний поток). Сетевые модели — единственные в каталоге без
схем; туториалов по сетям нет (есть только time_varying_load, power_of_two_choices
по смежным темам).

Первоисточники: Dallery–Frein (Operations Research 1993), Brandwajn–Jow (OR 1988),
Balsamo et al. «Analysis of Queueing Networks with Blocking» (Kluwer 2001);
fork-join в сетях — IEEE TPDS 2013 (doi:10.1109/tpds.2013.70), AAP 2004;
Gelenbe 1996 (мультикласс сигналов); Whitt–You 2019 (RQNA).

## Задачи

### П.1 Тандемы и сети с блокировками (конечные буферы)

- [x] `TandemBlockingCalc` (`theory/networks/blocking.py`): двухпроходная декомпозиция
      тандема с конечными буферами (Brandwajn–Jow / Dallery–Frein / Takahashi et al.),
      blocking-after-service; метрики: пропускная способность, загрузки, вероятности
      блокировки узлов и потерь на входе. Ключ к точности: в BAS потери только на
      входе — внутренние узлы получают сквозной поток X (без прореживания).
- [x] Симуляция: отдельный `TandemBlockingSim` (`sim/networks/tandem_blocking.py`) с
      каскадной разблокировкой, сидируемый — вместо инвазивного изменения общего
      `NetworkSimulator` (по образцу polling/closed: парный специализированный сим).
- [x] Валидация (`tests/test_tandem_blocking.py`, 4 теста): тандем 2 узла vs точная
      CTMC — **X в пределах 0.5%**, потери 3%; 3 узла vs сим (X 3%); буфер ∞ сводится
      к тандему Джексона (1e-9).

### П.2 Fork-join внутри маршрутизируемой сети

- [x] `OpenNetworkCalcForkJoin` (`theory/networks/fork_join_network.py`): fork-join
      станции в открытой сети, средние по декомпозиции; отклик FJ-узла — аппроксимации
      Varma / Nelson–Tantawi из существующего `ForkJoinMarkovianCalc`.
- [x] Симуляция: `ForkJoinNetworkSim` (`sim/networks/fork_join_network.py`) — обычные
      многоканальные узлы + fork-join узлы (k ветвей со своими очередями,
      синхронизация на join), сидируемый.
- [x] Валидация (`tests/test_fork_join_network.py`, 3 теста): изолированный FJ-узел ==
      `ForkJoinMarkovianCalc` (1e-12) и vs сим (5%); тандем M/M/2 → FJ(k=3) vs сим (8%).

### П.3 MAP-вход сети (непуассоновский внешний поток)

- [x] `NetworkSimulator.set_sources(..., source_kendall="MAP", source_params=...)` —
      непуассоновский внешний источник (MAP/H/E/... через `create_distribution`).
- [x] Хелпер `map_arrival_cv2(map_params)` в `theory/networks/qna.py`: (rate, c²)
      интервалов MAP для `OpenNetworkCalcQNA(arrival_cv2=...)`; ограничение
      (автокорреляция не учитывается — нижняя оценка перегрузки) задокументировано
      в docstring и в руководстве.
- [x] Валидация (`tests/test_map_network.py`): пачечный MMPP-2 (c²=3.8, сильная
      корреляция) на тандеме — QNA(c² MAP) **34%** vs QNA(Poisson) **61%**; остаток —
      цена игнорирования автокорреляции (честно показано, где ломается).

### П.4 Добор точности (резерв EPIC-017)

- [x] Многоканальные FCFS-станции в мультичейн-MVA (`BCMPClosedNetworkCalc`,
      параметр `n`) — маргинальные вероятности по векторам популяций (Bolch §8.2.2);
      одноклассовый случай совпадает с точным MVA до 1e-12, агрегация одинаковых
      классов точна (`tests/test_bcmp_network.py`, +2 теста).
- [x] `GNetworkMulticlassCalc` (Gelenbe 1996): PS-узлы, классовые сигналы; семантика
      убийства пришпилена численно — сигнал класса r убивает с вероятностью PS-доли
      k_ir/k_i (невязка глобального баланса 1e-6 против 1e-3 у «убивает всегда»).
      Валидация: сводимость к одноклассовому (1e-10) + невязка баланса точной CTMC
      (`tests/test_g_network.py`, +2 теста).
- [ ] ~~RQNA-поправки; mixed BCMP~~ — опциональные, не потребовались; остаются в
      резерве обзора.

### П.5 Transient/time-varying сети

- [x] `TimeVaryingNetworkCalc` (`theory/networks/time_varying_network.py`): PSA поверх

## Критерии готовности (DoD эпика)

Общий DoD ([../DOD.md](../DOD.md)). Специфично:

- Блокировки: расчёт vs точная CTMC на малых буферах в пределах нескольких процентов;
  buffer → ∞ сводится к открытой сети.
- Fork-join узел в изоляции совпадает с существующим fork-join расчётом/симулятором.
- MAP-вход: QNA(c² MAP) ближе к симуляции, чем QNA(Poisson), на коррелированном входе;
  ограничения задокументированы.
- Каждая схема — в обоих каталогах (EN+RU), в стиле существующих; туториалы
  выполняются end-to-end и добавлены в индекс.

## Результаты

Сетевой модуль доведён до практической полноты; все DoD-критерии выполнены:

- **`TandemBlockingCalc` + `TandemBlockingSim`** — тандемы с конечными буферами (BAS):
  пропускная способность в пределах 0.5% от точной CTMC; буфер ∞ сводится к Джексону.
- **`OpenNetworkCalcForkJoin` + `ForkJoinNetworkSim`** — fork-join станции внутри
  маршрутизируемой сети; изолированный узел совпадает с автономным fork-join (1e-12).
- **MAP-вход**: непуассоновский источник в `NetworkSimulator` + `map_arrival_cv2` для
  QNA; на сильно коррелированном MMPP ошибка 34% против 61% у пуассоновского допущения —
  граница применимости двухмоментного метода показана явно.
- **Добор точности**: многоканальные FCFS-станции в мультичейн-MVA (== точный MVA до
  1e-12); `GNetworkMulticlassCalc` — мультикласс сигналов Gelenbe 1996 с численно
  верифицированной семантикой убийства (невязка глобального баланса 1e-6).
- **`TimeVaryingNetworkCalc` + `TimeVaryingNetworkSim`** — PSA для сетей с λ(t);
  стационарный предел точен, медленная модуляция — 10% от сима.
- **Каталог**: 5 схем (EN+RU) в стиле EPIC-005 — у сетевых моделей впервые появились
  иллюстрации; раздел «Queueing networks» в обоих каталогах, руководство по сетям
  дополнено 4 разделами (EN+RU), README-строка Networks обновлена.
- **Туториалы**: 3 ноутбука с инсайтами (колено закрытой сети, Амдал для сетей,
  иллюзия пуассоновости), выполнены end-to-end, индекс туториалов обновлён (EN+RU).

Тесты: +14 новых (5 файлов + расширения BCMP/G-network); pylint новых модулей 10/10.
В резерве обзора остаются: RQNA, mixed BCMP, LQN, RL-управление
([обзор](../research/queueing-networks-2026.md), п. 2.6–2.8).
