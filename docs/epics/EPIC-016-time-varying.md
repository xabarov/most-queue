# EPIC-016: Нестационарные очереди Mt/M/c (PSA / MOL)

- **Статус:** done (2026-07-11)
- **Создан:** 2026-07-11
- **Roadmap:** обзор трендов — [../research/queueing-trends-2026.md](../research/queueing-trends-2026.md) (вторая волна, п. C)

## Цель

Добавить анализ **нестационарных** систем с зависящей от времени интенсивностью входа λ(t)
(суточные/сезонные профили) — аппроксимации **PSA (pointwise stationary approximation)** и
**MOL (modified offered load)** для нестационарных вероятности блокировки и задержки.

## Контекст

Реальный трафик нестационарен (пики нагрузки), а большинство формул предполагают стационар. Классика
transient/нестационарного анализа:
- **PSA**: подставить λ(t) в стационарную формулу — точно при медленных изменениях и большой ёмкости.
- **MOL**: сгладить λ(t) откликом M/M/∞ (offered load `m(t)`), затем стационарная формула Erlang —
  лучше на быстрых изменениях; ключ для staffing.
- Применение: колл-центры, облачный autoscaling, ЦОД с суточными профилями.

Первоисточники: Green–Kolesar–Whitt (staffing нестационарных очередей), Massey–Whitt (Mt/M/c),
см. research.

## Задачи

- [x] **`TimeVaryingMMcCalc`** (`theory/time_varying.py`): Mt/M/c — PSA и MOL для вероятности
      блокировки (Erlang B, `kind="loss"`) и ожидания (Erlang C, `kind="delay"`); вход λ(t) как
      callable, μ, c; MOL интегрирует ODE `dm/dt = λ(t) − μ·m(t)` с прогревом.
- [x] **`TimeVaryingMMcSim`** (`sim/time_varying.py`): неоднородный пуассоновский вход методом
      прореживания (thinning), система с потерями Mt/M/c/c, вероятность блокировки по фазам цикла;
      сидируемый.
- [x] Валидация: PSA/MOL vs симуляция для синусоидального λ(t). Медленно (period≈12.6):
      |PSA−sim|=0.029, |MOL−sim|=0.026. Быстро (period≈1.6): |PSA−sim|=0.099, |MOL−sim|=0.035 —
      MOL заметно точнее на быстрых изменениях, как и предсказывает теория. Стационарный предел
      (λ=const) = точный Erlang B. Тесты `test_time_varying.py` (6 шт.).
- [x] Каталог (EN+RU) + строка в сравнительной таблице.
- [x] Туториал `tutorials/time_varying_load.ipynb` — «пик загрузки не совпадает с пиком спроса»:
      отставание felt-load (arctan(w/μ)), затухание амплитуды, PSA vs MOL vs симуляция, рост ошибки
      PSA с частотой колебаний.

## Критерии готовности (DoD эпика)

Общий DoD ([../DOD.md](../DOD.md)). Специфично: при постоянном λ сводится к M/M/c точно; PSA/MOL
согласуются с симуляцией в своих режимах (медленные / быстрые изменения).

## Результаты

- **`TimeVaryingMMcCalc`** — PSA (поточечная стационарная) и MOL (сглаженная нагрузка через отклик
  M/M/∞) для вероятности блокировки/ожидания при λ(t); **`TimeVaryingMMcSim`** — парный симулятор
  (неоднородный Пуассон через thinning, система с потерями). MOL точнее PSA на быстрых изменениях
  (0.035 против 0.099), обе сходятся к точному Erlang B при постоянном λ. Каталог EN+RU, тесты (6 шт.),
  туториал `time_varying_load.ipynb`.
