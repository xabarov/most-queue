# Gap-анализ: модели теории очередей vs реализованные в Most-Queue

> Результат задач 2–3 эпика [EPIC-001](epics/EPIC-001-queueing-models-gap-analysis.md).
> Дата: 2026-07-08. Источники верифицированы через OpenAlex/Crossref/arXiv (скилл `lit-search`)
> и веб-поиск; DOI указаны там, где они существуют.

## 1. Метод

Ландшафт моделей ТМО сопоставлен с кодом `most_queue.theory` / `most_queue.sim` по четырём
направлениям: базовые модели, средний уровень, продвинутые методы, экосистема open-source
инструментов (для оценки дифференциации). Критерии приоритета:

1. **Ценность** — прикладной спрос + научная значимость;
2. **Сложность** — с учётом переиспользования имеющегося ядра (моментная машинерия M/G/1,
   busy-period LST, Такахаси-Таками, симулятор `QsSim`);
3. **Дифференциация** — есть ли уже хорошая open-source реализация;
4. **Стратегический фит** — соответствие научному фокусу проекта
   (ML×очереди, scheduling с предсказаниями, retrial, MAP/PH, матрично-аналитические методы —
   см. `works/queueing_systems_review/general.md`).

## 2. Выводы обзора экосистемы (кратко)

- **Уникальные ниши most_queue уже сейчас**: size-based аналитика (SRPT/SJF/PSJF/SPJF) — open-source
  конкурентов нет; vacation-модели, negative customers (G-queues), retrial — аналитических
  реализаций нет ни в одном maintained-пакете; Такахаси-Таками для M/PH/n — редкость.
- **Вакуум MAP/PH**: эталонный BuTools заморожен с 2016 г. (без лицензии), SMCSolver/Q-MAM —
  академический MATLAB. Поддерживаемый pip-устанавливаемый стек QBD/MAP/PH — открытая ниша.
- **Коммодити (не конкурировать, но иметь для полноты)**: базовые M/M/c-калькуляторы, Erlang B/C,
  MVA/закрытые сети (JMT, LINE, R `queueing`), генерические DES (Ciw, simmer, SimPy).
- Ближайший аналог по замыслу «аналитика+симуляция в одном пакете» — LINE Solver (Java, BSD-3,
  активный); дифференциация most_queue — Python-native и семейства моделей, которые LINE
  аналитически не решает.

## 4. Обоснование по волнам

### Волна 1 — «must»: быстрые точные победы (суммарно ~2–3 недели)

Все семь пунктов — точные формулы, ложащиеся на готовую машинерию:

- **Erlang B/C, M/M/∞, M/G/∞** — table stakes: их ожидают от любой библиотеки ТМО.
  Erlang B через устойчивую рекурсию B(n,a); нечувствительность M/G/n/0 — Севастьянов (1957),
  код бесплатен. M/G/∞: стационарное число заявок — Пуассон(λE[S]), нечувствительность.
- **GI/G/1/GI/G/m аппроксимации** — двухмоментные формулы, прямой фит в moment-based API;
  валидация готовым симулятором GI/G/n. Самый частый практический запрос (capacity planning).
- **Классические vacations** — моментная арифметика поверх P-K (`fifo/mg1.py`); даёт точный
  baseline для существующих H2-аппроксимаций прогрева/охлаждения.
- **PS + FB/LAS** — завершают историю дисциплин: size-aware (SRPT/SJF/SPJF — уже есть) vs
  blind (PS, FB). FB переиспользует утилиты усечённых моментов `theory/srpt/` почти дословно.
  LCFS-PR: sojourn = busy period M/G/1 — рекурсии Такача уже в `theory/utils/busy_periods`.
- **Ненадёжный прибор** — сводится к M/G/1 с generalized service (completion time),
  структурно = vacation-декомпозиция; закрывает семейство «прибор недоступен»
  (vacations ✓, negatives ✓, breakdowns —).

### Волна 2 — «should»: стратегические направления

- **QBD/MAP/PH-стек — флагман.** Наивысшая научная ценность (коррелированный трафик, где
  пуассоновские модели ошибаются на порядки), вакуум после заморозки BuTools, прямой фит
  в научный фокус проекта и в имеющуюся фазовую машинерию Такахаси-Таками. Ядро — R-матрица
  через logarithmic reduction (Latouche–Ramaswami 1993, ~100 строк numpy); M/PH/1, PH/PH/1,
  MAP/PH/1 — один QBD-кернел; MMPP — «cookbook» Fischer–Meier-Hellstern. Заодно QBD даёт
  альтернативный точный решатель для кросс-валидации существующих M/H2/n.
- **Erlang-A** — многоканальное обобщение `impatience/mm1.py`, birth-death с точными формулами,
  колл-центры/staffing — крупнейшая прикладная аудитория.
- **Retrial** — точные PGF/LST-решения (Falin–Templeton), стыкуется с фокусом на
  retrial+MAP (Vishnevsky и др.); в симуляторе нужен механизм орбиты.
- **MVA/Buzen** — коммодити, но дёшево и закрывает самый заметный функциональный пробел
  (закрытых сетей нет вообще); один класс — дни работы.
- **Batch-обслуживание** — симметрично дополняет batch arrivals; M/M^Y/1 — корень полинома
  + геометрический хвост.

### Волна 3 — «could» и «defer»

QNA — крупный, но осмысленный апгрейд сетевого модуля (непуассоновские внутренние потоки).
PSA/Halfin–Whitt — тонкие обёртки, удобный «бантик» к staffing-сценариям. BMAP/G/1 — вторая
фаза MAP-направления (M/G/1-type, существенно тяжелее QBD). Polling и дискретные очереди
отложены: целые подобласти с малым переиспользованием ядра и нишевой аудиторией.
Многосерверный SRPT — открытая исследовательская тема (Grosof–Scully–Harchol-Balter 2018,
Open Problem в `works/.../SRPT/`), кандидат в научную работу, а не в библиотечную фичу.

## 6. Рекомендации

1. Реализовать волну «must» одним-двумя релизами — семь точных моделей малой сложности,
   закрывающих базовые ожидания и усиливающих сильные стороны (дисциплины, vacations).
2. Завести отдельный эпик/roadmap на **QBD/MAP/PH-стек** — флагманское направление
   с наибольшей научной отдачей и явной нишей после заморозки BuTools.
3. Завести roadmap на **retrial + Erlang-A** (объединяет прикладной спрос и научный фокус).
4. MVA/Buzen добавить как утилитарную полноту (не флагман).
5. Polling, дискретные очереди, BMAP/G/1 — отложить до закрытия волн 1–2;
   многосерверный SRPT — рассматривать как исследовательскую статью на базе библиотеки.
