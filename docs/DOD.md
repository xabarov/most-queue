# Definition of Done (DoD)

Критерии готовности работ в Most-Queue. Задача/эпик считается завершённой только при выполнении
соответствующего чек-листа. См. также [PROJECT.md](PROJECT.md) и [INFRASTRUCTURE.md](INFRASTRUCTURE.md).

## DoD: новая модель СМО (калькулятор)

- [ ] Класс наследует `BaseQueue` и следует контракту `set_sources` / `set_servers` / `run`,
      возвращает стандартную структуру результатов (`most_queue/structs.py`).
- [ ] Указан первоисточник метода (статья/книга) в docstring и в документации.
- [ ] Есть проверка условий применимости (стабильность ρ < 1, диапазоны параметров) с понятными ошибками.
- [ ] Есть тест сверки с симуляцией (или с известным аналитическим частным случаем, если
      симулятора для модели нет), допуски — в `tests/default_params.yaml`.
- [ ] Модель добавлена в каталог `docs/models.md` (описание, класс, пример кода).
- [ ] При нетривиальном методе — методический документ в `docs/` (по образцу `srpt_spjf_methods.md`).

## DoD: новый симулятор / расширение симуляции

- [ ] Совместим с общим интерфейсом (`set_sources`, `set_servers`, `run(num_of_jobs)`),
      собирает стандартную статистику (моменты w/v, вероятности состояний, utilization).
- [ ] Валидация: сходится к известному аналитическому результату на частном случае
      (например, M/M/1) и/или сверен с калькулятором.
- [ ] Тест в `tests/` (быстрый вариант; длинные прогоны — под маркером `slow`).
- [ ] Описание в `docs/simulation.md` и/или `docs/models.md`.

## DoD: исследовательская задача (обзор, gap-анализ)

- [ ] Результат оформлен документом в `docs/` (или `works/` для черновых обзоров)
      с указанием источников.
- [ ] Выводы actionable: список кандидатов с оценкой сложности/ценности, рекомендации
      «что делать дальше».
- [ ] Эпик/roadmap обновлён по итогам (закрыты вопросы, заведены следующие задачи).

## DoD: любая задача с кодом

- [ ] `pytest tests/ -m "not slow"` проходит локально; затронутые slow-тесты прогнаны отдельно.
- [ ] Код отформатирован: `black` + `isort` (line-length 120); `pylint most_queue` без новых замечаний.
- [ ] Публичные классы/методы имеют docstring на английском (как в существующем коде).
- [ ] Документация (`docs/`) и при необходимости README обновлены.
- [ ] Изменения не ломают обратную совместимость публичного API без явного решения
      (иначе — фиксация в описании релиза и повышение минорной/мажорной версии).

## DoD: эпик

- [ ] Все задачи эпика закрыты или явно перенесены (с указанием куда).
- [ ] Итог зафиксирован в файле эпика (раздел «Результаты»), статус переведён в `done`.
- [ ] Документация и roadmaps синхронизированы с фактическим состоянием кода.
