## Что это

**Most-Queue** — Python-библиотека для расчёта и имитационного моделирования систем массового
обслуживания (СМО) и сетей очередей. Публикуется на PyPI как `most-queue` (текущая версия — см.
`pyproject.toml`). Автор и мейнтейнер: Роман Хабаров (xabarov). Лицензия MIT.

Ключевая идея библиотеки: для каждой модели СМО, где возможно, есть **пара** — численный
калькулятор (`most_queue.theory`) и симулятор (`most_queue.sim`), а тесты сверяют их результаты
между собой (в терминах моментов времени ожидания/пребывания и вероятностей состояний).

## Структура репозитория

```

  theory/            — численные/аналитические методы
    fifo/            — базовые FIFO-модели (M/M/c/r, M/G/1, GI/M/1, GI/M/c, M/D/c, Ek/D/c,
                       Такахаси-Таками: M/H2/c, H2/M/c, H2/H2/c)
    priority/        — приоритетные системы (preemptive/non-preemptive, инвариантная аппроксимация)
    vacations/       — системы с прогревом/отпусками/задержкой
    negative/        — отрицательные заявки (RCS, disaster)
    networks/        — сети очередей (в т.ч. с приоритетами и отрицательными заявками)
    fork_join/       — Fork-Join / Split-Join
    batch/           — пакетное поступление (M^x/M/1)
    closed/          — закрытые системы (Engset)
    impatience/      — нетерпеливые заявки
    srpt/            — size-based дисциплины (SRPT, SJF, PSJF, SPJF + предикторы)
    utils/           — общие численные утилиты (свёртки, преобразования, моменты)
    base_queue.py    — базовый класс калькуляторов (контракт set_sources/set_servers/run)
    calc_params.py   — параметры численных методов (точность, размеры сеток и т.п.)
  sim/               — имитационное моделирование
    base.py, base_core.py — базовый симулятор QsSim (каналы, дисциплины, сбор статистики)
    priority.py      — PriorityQueueSimulator
    size_based.py    — SizeBasedQsSim (SRPT/SJF/PSJF/SPJF)
    negative.py, vacations.py, batch.py, impatient.py, finite_source.py,
    fork_join.py, fork_join_delta.py, flow_sum.py — специализированные симуляторы
    networks/        — симуляторы сетей
  random/            — распределения (H2, Erlang, Gamma, Pareto, Normal, Deterministic, ...):
                       генерация, моменты, подгонка параметров по mean/CV (в т.ч. complex-fit H2)
  io/                — вывод результатов (таблицы, печать)
  misc/              — вспомогательное
  structs.py, constants.py — общие структуры результатов и константы

tests/               — интеграционные тесты «теория vs симуляция» + units/
tutorials/           — Jupyter-ноутбуки с примерами
examples/            — примеры кастомных симуляторов и сетей
docs/                — документация (RU) + dev-документы + roadmaps/ + epics/
works/               — исследовательские материалы: обзоры литературы, статьи, эксперименты
                       (не входит в пакет; источник теории для новых моделей)
```

## Соглашения и паттерны

- **Контракт калькулятора**: класс наследует `BaseQueue`, конфигурируется через
  `set_sources(...)` и `set_servers(...)`, запускается `run()`, возвращает структуру результатов
  (см. `most_queue/structs.py`) с моментами `w` (ожидание), `v` (пребывание), вероятностями `p`,
  загрузкой `utilization`.
- **Моменты, а не только средние**: методы, как правило, считают несколько начальных моментов;
  распределения задаются либо параметрами, либо списком начальных моментов.
- **CV < 1 через complex-fit H2**: гиперэкспоненциальная аппроксимация с комплексными
  параметрами — стандартный приём для Такахаси-Таками; симулятор комплексные параметры
  не генерирует, валидация — через эквивалентную по mean/CV Gamma.
- **Тест = сверка теории с симуляцией**: типовой тест задаёт параметры из
  `tests/default_params.yaml`, запускает калькулятор и симулятор, сравнивает с допуском.
- Стиль: black + isort, line-length 120; pylint с ослаблениями (см. `pyproject.toml`).
- Язык документации — русский; README в корне — английский (основной, попадает на PyPI),
  русская версия — `README.ru.md` (при правках синхронизировать обе).

## Где искать теорию

- `docs/` — методические описания реализованных методов (Такахаси-Таками, SRPT/SPJF, negative и т.д.).
- `works/queueing_systems_review/` — конспекты первоисточников.
- `docs/статьи/`, `works/articles/` — PDF первоисточников.
- `docs/roadmaps/` — планы по конкретным направлениям (образец: `srpt_spjf_roadmap.md`).
- Поиск новой литературы — скилл `lit-search` (`.claude/skills/lit-search/`): опрос
  arXiv + OpenAlex + Crossref одним скриптом со сводной таблицей по цитируемости.
